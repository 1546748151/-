邻选 · 部署使用指南

按照以下步骤，大约需要 15 分钟 完成全部设置。全程免费，无需购买服务器或域名。

第一步：注册 Supabase 账号

1

Supabase 是一个免费的开源数据库服务，我们用它来存储商品和订单数据。

  1. 用浏览器打开 supabase.com
  2. 点击右上角 "Start your project"
  3. 选择 "Sign in with GitHub"（推荐，最方便）
  4. 如果没有 GitHub 账号，也可以用邮箱注册
免费版提供 500MB 数据库空间和每月 5GB 流量，对于小区团购来说绰绰有余。

第二步：创建数据库项目

2
  1. 登录后点击 "New project"
  2. 填写以下信息：
    • Name：填 groupbuy（或任意名字）
    • Database Password：设置一个数据库密码（建议用浏览器生成的随机密码，记下来备用）
    • Region：选择 Northeast Asia (Tokyo) 或 Southeast Asia (Singapore)，离国内最近，速度最快
  3. 点击 "Create new project"，等待 1-2 分钟让数据库创建完成

第三步：初始化数据库表

3

在 Supabase 项目面板中，左侧菜单找到 SQL Editor，执行以下操作：

  1. 点击 "New query"
  2. 打开 supabase-schema.sql，复制全部内容
  3. 粘贴到 SQL Editor 中
  4. 点击右下角绿色 "Run" 按钮
执行成功后，你会看到三个表：products（商品）、orders（订单）、settings（设置），以及自动创建的安全策略。这意味着数据库已经准备就绪。

第四步：获取连接信息

4

在 Supabase 项目面板左侧菜单，找到 Settings > API：

  1. 复制 Project URL（格式如 https://xxxxx.supabase.co）
  2. 复制 anon public key（一长串以 eyJ 开头的文字）
  3. 把这两个信息保存到记事本备用，下一步用到
请使用 anon public key（公开密钥），不要使用 service_role key（私密密钥）。service_role key 会绕过所有安全策略，不应该在网页中使用。

第五步：部署网页到 Cloudflare Pages

5

Cloudflare Pages 是免费、快速的静态网页托管服务，而且在国内也能正常访问。

  1. 用浏览器打开 dash.cloudflare.com
  2. 注册/登录 Cloudflare 账号（可用邮箱注册）
  3. 左侧菜单找到 Workers & Pages，点击进入
  4. 点击 "Create" → 选择 "Pages" → 选择 "Upload assets"
  5. 给你的项目起个名字（如 groupbuy）
  6. 把以下三个文件拖拽上传：
    • order.html — 核心下单工具
    • landing.html — 宣传页面（可选）
    • supabase-schema.sql — 数据库脚本（备存）
  7. 点击 "Deploy"
  8. 等待 1 分钟左右部署完成，你会得到一个网址如 https://groupbuy.pages.dev
这个网址就是你的下单页面地址。邻居扫码或点击这个链接就能进入下单页面。

备选方案：如果没有 Cloudflare 账号

你也可以使用其他免费托管服务：

  • GitHub Pages：把文件上传到 GitHub 仓库，在 Settings 中启用 Pages
  • Vercel：类似 Cloudflare Pages，注册后拖拽部署
  • Netlify：同样支持拖拽部署静态页面

第六步：首次使用配置

6

用浏览器打开你刚部署的网址，会看到初始化设置页面。

  1. 在 Supabase URL 输入框粘贴第四步复制的 Project URL
  2. 在 Anon Key 输入框粘贴第四步复制的 anon key
  3. 点击 "保存并连接"

连接成功后会自动进入顾客下单页面。

然后设置管理员密码：

  1. 在网址后面加上 #admin（如 https://groupbuy.pages.dev#admin），回车
  2. 这是首次进入，直接输入你想设置的管理密码并回车（密码就设置好了）
  3. 进入后台后，点击 "设置" 标签可以修改密码
请务必记住你的管理员密码。如果忘记了，需要在 Supabase 的 SQL Editor 中手动清空 admin_password 记录来重置。

第七步：日常使用教程

发起一次团购的完整流程

1. 进入后台：在浏览器打开你的网址，在网址后加 #admin，输入管理密码。

2. 添加商品：在"商品"标签页，点击"添加商品"：

  • 填写商品名称（如"烟台红富士苹果"）
  • 可选填写商品图片链接（在微信中长按图片 → 复制图片链接）
  • 添加规格：如"5斤装 / 28元"、"10斤装 / 50元"
  • 可选设置库存数量（卖完自动停售）
  • 点击"添加商品"

3. 生成下单二维码：

  • 点击"二维码"标签页
  • 页面自动显示你的下单二维码
  • 长按二维码图片 → 保存到手机相册
  • 把二维码发到业主微信群，配上文字说明（如"本周团购开始啦，大家扫码下单"）

4. 设置截止时间（可选）：

  • 点击"设置"标签页
  • 设置下单截止时间
  • 到了截止时间，顾客端会自动显示"下单已结束"

5. 查看订单：

  • 点击"订单"标签页，实时查看所有邻居的下单
  • 每个用户的订单自动按人汇总，显示购买明细和应付金额
  • 底部显示每个商品的总销量统计
  • 点击"导出 Excel"可以下载订单表格用于备货对账

6. 收款：

  • 截止后，点击"收款"标签页
  • 页面列出所有人的应付金额
  • 点击"一键复制收款清单"
  • 打开微信 → 业主群 → 群工具 → 群收款
  • 参考复制的清单，逐一发起群收款

顾客如何使用

  1. 在微信群里长按二维码 → 识别图中二维码
  2. 打开页面后，看到所有在售商品
  3. 点击商品选择规格和数量
  4. 选好后点击底部"去下单"
  5. 填写群昵称（必填）+ 备注（选填）
  6. 提交订单，完成
  7. 如需修改，点击页面上方"查/改订单"，输入昵称查询并操作
温馨提示：Supabase 和 Cloudflare Pages 的免费额度对于一个小区的团购来说完全足够。如果未来你的团购规模扩大，Supabase 的付费方案也只要 $25/月起步。但大概率你用不到付费方案。
